Running squid
Squid is normally run from a script that re-starts it in the (normallyminimal) chance that it dies. There are 2 versions of this script: /usr/local/squid/bin/RunCache and /usr/local/squid/bin/RunAccel For more info on using Squid as an HTTP accelerator (and thus using the RunAccel script) look at this section

The RunCache script normally runs in the foreground, so to startsquid, type the following:

% /usr/local/squid/bin/RunCache& %

then look at the main squid log file /usr/local/squid/logs/cache.log with something along the lines of the follwing command:

% more /usr/local/squid/logs/cache.log

You should see messages about squid starting, along the lines of thefollowing: 

      97/02/24 19:34:52| Starting Squid Cache version 1.1.10 for i586-pc-linux-gnu...97/02/24 19:34:52| With 256 file descriptors available97/02/24 19:34:52| Initializing IP Cache...

If you don't see any messages about FATAL errors, squid should berunning happily on your machine now.

Configure your browser as described in the next section and you can give ita try!
